# Python Engine — AI Cost Calculator

A pure Python 3.9+ port of `public/lib/cost-engine.js`. Produces identical
numbers to the JS reference implementation (verified at `rel_tol=1e-9`,
`abs_tol=1e-6` across all 17 bundled presets).

## Structure

```
python/
  costcalc/
    __init__.py       exports compute()
    prices.py         loads prices_data.json; exposes Prices, rate cards, GPU catalog
    workload.py       normalize_workload(), compute_queries()
    llm.py            per-query token math, cache helpers, compute_api_cost()
    agents.py         per_query_cost_agents() — multi-agent pipeline
    tools.py          compute_tool_fees()
    verification.py   compute_verification() — NLI/selfCheck/flatPerCheck + cascade
    federal.py        compute_federal(), compute_ato_from_prices()
    selfhost.py       compute_self_host(), compute_hybrid()
    fixed.py          compute_fixed_costs(), resolve_infra_cost()
    extras.py         compute_embedding(), compute_personnel(),
                      compute_reservation(), compute_agent_engineering()
    headline.py       compose_headline()
    engine.py         compute() — top-level orchestrator
    prices_data.json  serialized Prices object (generated by dump-prices.mjs)
  parity_check.py     acceptance gate (deep numeric diff vs JS dumps)
  run.py              CLI runner
```

## Dependencies

Standard library only: `json`, `math`, `pathlib`, `argparse`. No pip install needed.

## Usage

**CLI:**
```bash
python3 python/run.py public/examples/public-geospatial-qa.json
python3 python/run.py public/examples/swe-bench-coding-agent.json --hosting api --json
python3 python/run.py public/examples/voice-support-agent.json --quiet
```

**Library:**
```python
import json, sys
sys.path.insert(0, "python")
from costcalc import compute

workload = json.load(open("public/examples/public-geospatial-qa.json"))
opts = {
    "hosting": "api",
    "model": "gpt-5.2",
    "tier": "standard",
    "mix": "mixed",
    "costMode": "optimistic",
    "botFactor": 1.5,
    "cacheRate": 0.88,
    "verifCoverage": 0.1,
}
result = compute(workload, opts)
print(f"API: ${result['api']['monthly_with_retry']:,.2f}/mo")
print(f"Headline: ${result['headline']['headline']:,.2f}/mo")
```

## Parity Verification

The acceptance gate compares all 17 preset results against pre-dumped JS
engine output (from `scripts/dump-engine.mjs`). Verified numeric leaves:
`queries`, `api`, `verification`, `tool_fees`, `federal`, `reservation`,
`embedding`, `personnel`, `fixed_costs`, `self_host`.

```bash
# Regenerate JS dumps (only needed after engine changes):
node scripts/dump-engine.mjs

# Run parity check:
npm run test:parity-py
# or directly:
python3 python/parity_check.py
```

## Options (compute() second argument)

| Key | Default | Description |
|-----|---------|-------------|
| `hosting` | `"api"` | `api`, `self`, `hybrid`, `onprem` |
| `model` | workload default | Model ID from rate_cards |
| `tier` | `"standard"` | Tier multiplier key |
| `mix` | `"mixed"` | Shape mix key |
| `costMode` | `"realistic"` | Self-host cost mode |
| `botFactor` | `1.5` | Bot traffic multiplier |
| `cacheRate` | from workload | Baseline cache hit rate (0–1) |
| `verifCoverage` | `0` | Verification coverage (0–1) |
| `batchShare` | `0` | Fraction routed through batch API |
| `langMult` | `1` | Language token multiplier |
| `contextCompressionPct` | `0` | Context compression savings (0–0.7) |
| `retryInflate` | auto | Retry cost multiplier; auto = 1 + 1.5×retry_rate |
| `apiSplit` | `0.5` | Hybrid API fraction (hybrid only) |

## Key Implementation Notes

- `effective_cached_rate()`: when `cached_write_per_million` is absent from a
  rate card, returns `p_read` unconditionally (OpenAI auto-prefix assumption).
  This is NOT the same as blending with write share.
- Daily-cap ordering: `host_mult` is applied **before** capping, then divided
  back out to produce `monthly_capped_pre_federal`.
- Tool token split (fix-A): schema tokens are cache-eligible at the agent's
  cache rate; result tokens use `tool_result_cache_share` (default 0.5).
- Agent mix blending: agent costs are blended across shape mix weights (same
  as workload mode). Presets without a mix fall back to `{full: 1.0}`.
- `TURN_ADJ_BOUND = 0.15` (not the paper's [0.50, 0.94] full clamp).


## Deliberate divergences from the JS engine

- **Absent `anchor_query.cache_rate_baseline` (and no `opts['cacheRate']`)** —
  the JS engine reads `undefined` and the token math silently produces NaN.
  The Python engine refuses to reproduce silent garbage and raises
  `ValueError` instead. Every shipped preset sets the field, so this only
  fires on hand-built minimal workloads. (Found by adversarial fuzz audit,
  2026-06-11: 1,276 perturbed variants, zero other divergences.)

## Randomized cross-check

Beyond the 17-preset parity gate, `random_parity.py` generates seeded
random workload mutations (segments, tools, verification, caps,
reservations, embeddings, agent fields, models/tiers/hostings) and runs
every case through BOTH engines, deep-comparing at the same tolerance:

```bash
python3 python/random_parity.py --n 1000 --seed 7
```

Verified at port time: 2,300 random cases across seeds 1/7/42 — all
match, zero asymmetric crashes.
